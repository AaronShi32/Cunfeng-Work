## AI Agent 架构与开发

### Q1：什么是 AI Agent 的 "Skill"（技能）？如何设计一个高质量的技能？

**核心定义**：Skill 是 Agent 执行特定任务的原子能力（如搜索、发邮件、读文件）。它将 Python 函数逻辑封装成 LLM 可识别的工具。

**设计要素**：

- **精确的 Docstring**：LLM 通过解析函数描述、参数说明来决定是否调用。描述应包含"什么时候用"和"参数代表什么"。
- **输入验证**：在代码层对模型传入的参数进行强校验，防止 LLM 传错格式。
- **结构化输出**：返回给模型的结果应易于解析，失败时需返回明确的错误原因。

### Q2：为什么说 "Skill" 是解决 LLM 幻觉的关键工程手段？

- **确定性**：逻辑归代码（Python），意图归模型。复杂的数学计算或数据库查询由代码执行，结果是 100% 准确的。
- **边界感**：通过 Skill 限制了 Agent 的权限（如只允许读取特定 NAS 目录），防止模型在无法触达的领域"一本正经地胡说八道"。

## 协议、集成与 RAG 系统

### Q3：什么是 MCP (Model Context Protocol) 协议？它解决了什么痛点？

**核心定义**：由 Anthropic 推出的开放协议，旨在统一 AI 模型与外部数据源（如本地文件、GitHub、Google Drive）的连接方式。

**解决痛点**：

- **标准化**：不再需要为每个 Agent 框架写独立的插件。
- **安全性**：MCP 作为一个中介服务器，可以更好地管理本地私有数据（如 NAS 上的文档）的访问权限。

### Q4：在本地 NAS 环境下，如何优化 RAG（检索增强生成）的精度？

- **挂载层**：利用 SMB/NFS 将 NAS 映射为本地路径，减少网络延迟。
- **混合检索 (Hybrid Search)**：结合向量检索 (Semantic Search) 和关键词检索 (BM25)。向量负责捕捉语义，关键词负责匹配特定名词（如文件名、项目代码）。
- **切片优化**：根据文档类型（Markdown, PDF）选择合适的 Chunk Size（切片大小），并保持一定的 Context Overlap（上下文重叠）。

## 深度学习原理与模型微调

### Q5：解释 Transformer 中的 Self-Attention 机制及其计算瓶颈。

**核心逻辑**：通过计算 Q (Query), K (Key), V (Value) 矩阵，让模型在生成当前 Token 时能"看到"整个序列的相关信息。

**公式**：Attention(Q, K, V) = softmax(QK^T / √d_k) · V

**瓶颈**：其计算复杂度与显存占用随序列长度 N 呈平方级增长 O(N²)。

**优化**：Flash Attention（减少 IO）、GQA（分组查询注意力）等是目前主流的优化手段。

### Q6：什么是 LoRA (Low-Rank Adaptation)？它对个人开发者有何意义？

**核心原理**：在原始模型参数旁添加两个低秩矩阵进行训练，冻结主模型权重。

**意义**：

- **低资源需求**：在 Mac mini M4 (16GB+ 内存) 上即可微调 7B/14B 模型。
- **快速切换**：训练出的 Adapter 极其轻量（通常几十 MB），可以针对不同任务（如代码编写、财务分析）快速热插拔。

## 工程化落地与硬件加速

### Q7：针对 Apple Silicon (M4 芯片)，如何榨干硬件性能进行 AI 推理？

- **统一内存利用**：由于 CPU 和 GPU 共享内存，应优先选择支持 Metal 加速的框架（如 MLX, llama.cpp）。
- **量化技术 (Quantization)**：使用 4-bit 量化（GGUF/EXL2）可降低 70% 的内存占用，同时精度损失极小。
- **无 UI 部署 (Headless)**：通过 Docker 运行本地模型服务（如 Ollama），减少系统 GUI 对显存的占用。

### Q8：如何处理本地 AI 环境中的数据隐私与安全？

- **完全离线化**：使用 Ollama 或 LM Studio 运行本地模型，Skill 调用不经过公网 API。
- **容器隔离**：将 Agent 运行在 Docker 容器中，限制其对宿主机文件系统的访问权限，仅挂载必要的 NAS 路径。
